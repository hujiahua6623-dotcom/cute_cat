# Contributing Guide

感谢你愿意参与 `cute_cat` 项目。

## 如何贡献

1. 先阅读：
   - `README.md`
   - `doc/项目设计文档.md`
   - `doc/切片0-2工程骨架.md`
2. 提交 Issue 说明问题/想法（Bug、需求、设计讨论均可）。
3. Fork 并创建分支开发：
   - `feat/<short-name>`
   - `fix/<short-name>`
4. 提交 Pull Request，描述：
   - 变更内容
   - 变更原因
   - 测试方式

## 提交建议

- 保持单一职责：一个 PR 只做一件事
- 优先可读性与可维护性
- 避免提交无关格式化或大范围无意义改动

## 代码与设计约定

- 后端是游戏逻辑权威，前端不应绕过规则直接改核心状态
- AI 输出仅限受控字段（文案/建议/记忆），不得直接改核心数值规则
- 与时间相关的逻辑必须遵守设计文档中的换算与“离线不暂停”原则

## PR 检查清单（建议）

- [ ] 已阅读相关文档并与当前设计一致
- [ ] 变更有基本测试或验证说明
- [ ] 未引入明显破坏性改动
- [ ] 文档已同步更新（如涉及行为变化）
