# 周期 5 发布前检查单（One-Page）

> 目标：在宣告“可对外演示/小范围试用”前，用一页清单完成可追溯门禁。

## A. 环境与启动（必过）

- [ ] 按 `infra/README.md` 完成 `docker compose up -d --build`
- [ ] `docker compose ps` 显示 `mysql` / `backend` / `frontend` 均为 `Up`（含 health）
- [ ] `GET /health` 与 `GET /api/v1/health` 均返回正常
- [ ] 关键环境变量完成安全替换（`JWT_SECRET`、MySQL 密码、CORS）

## B. 核心玩法回归（必过）

- [ ] 注册 -> 领宠 -> 进入花园（无阻塞）
- [ ] `joinGarden -> petAction -> petStateDelta` 可复现
- [ ] 至少一次 `Feed`、`Cuddle`、`Pat` 都有可见反馈
- [ ] 离线摘要可触发并可关闭

## C. 库存一致性回归（必过）

- [ ] `shop/buy` 后客户端库存即时更新（`inventoryChanged`）
- [ ] `Feed` 后客户端库存即时扣减（`inventoryChanged`）
- [ ] 刷新或短时断线后库存最终与 `/shop/inventory` 一致

## D. AI 双路径回归（必过）

- [ ] AI 可用（配置 `DASHSCOPE_API_KEY`）时，事件/治疗建议正常展示
- [ ] AI 不可用（清空 `DASHSCOPE_API_KEY`）时，fallback 文案生效且玩法不受阻
- [ ] 两种路径均有可追溯命令结果与截图

## E. 安全与稳定性基线（必过）

- [ ] WebSocket 入站限流触发正常（`TOO_MANY_REQUESTS`）
- [ ] 非法消息/非法坐标被拒绝（`BAD_REQUEST`）
- [ ] 后端日志可定位关键行为（join/action/rate-limit/bad-message）

## F. 证据留痕（必过）

- [ ] `doc/开发进度日志.md` 当日小节记录：命令、结果、截图路径、结论
- [ ] 至少包含以下命令结果：
  - [ ] `cd backend && PYTHONPATH=src pytest -q`
  - [ ] `cd frontend && npm run build`
  - [ ] `cd frontend && node scripts/capture-screenshots.mjs`
- [ ] 冷启动模板已回填：`infra/cold-start-validation-template.md`（复制到日志）

## G. 发布判定

- [ ] 通过：进入“可对外演示/小范围试用”
- [ ] 不通过：记录阻塞项、Owner、修复截止时间
