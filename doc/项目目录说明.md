## 项目目录说明（cute_cat）

用于记录当前项目目录结构与各文件/资源的用途，便于前端与后端协作实现。

### 根目录
1. `AGENTS.md`
   - 新会话 Agent 入口文档：快速理解项目背景、技术栈、关键规则、优先阅读文档与首个实现目标。
2. `doc/`
   - 项目设计与规格文档存放位置。
3. `assets/`
   - 静态资源（当前主要是 UI 像素风 mockup 图）。

### `doc/`
1. `doc/项目设计文档.md`
   - 项目完整设计说明：MVP 边界、多人生存的花园机制、宠物状态机与成长规则、经济系统、事件模板、AI 受控边界、时间连续推进与离线不暂停等。
2. `doc/切片0-2工程骨架.md`
   - 首个实现阶段（切片 0 + 2）的模块拆解、协议口径和最小验收清单。
3. `doc/开发收尾规范.md`
   - 每日/每次会话结束时的统一动作规范（日志记录、技术栈同步、文档联动更新）。
4. `doc/开发进度日志.md`
   - 每日进度追踪、风险记录与下一步计划。
5. `doc/开发周期计划.md`
   - 从周期 0 到周期 5 的分阶段迭代路线、各周期验收标准、时间盒参考；含 UI/前端/后端/测试/运维协作分工说明。
6. `doc/后端开发设计文档.md`
   - 后端技术架构、技术选型（MySQL、Redis、DashScope/LangChain、JWT）、模块划分、环境变量与非功能需求。
7. `doc/API-后端与前端对接.md`
   - HTTP（JWT + REST）与 WebSocket 消息字段、JSON 示例及动作枚举（与切片协议一致）。
8. `doc/UI设计-规范与稿清单.md`
   - UI token、参考稿清单与 API 字段对照（与 [doc/前端开发任务与对接.md](前端开发任务与对接.md) §10 配套）。
9. `doc/运维记录-仓库与首次推送.md`
   - Git 仓库初始化、与 GitHub 远端关联及首次推送的运维留痕与操作说明（凭据需在本机配置）。

### 说明
- macOS 生成的系统文件（如 `.DS_Store`）无需纳入实现流程，可忽略。
- **后端已实现**：`backend/src/cute_cat/`（FastAPI、`api/`、`auth/`、`game/`、`persistence/`、`realtime/`、`services/`）；迁移在 `backend/alembic/`；说明见 [`backend/README.md`](../backend/README.md)。
- **前端**：`frontend/` 已初始化（`Vite + TypeScript` + `Phaser 3`）；源码在 `frontend/src/`（`network/`、`state/`、`game/`、`styles/`，入口 `main.ts`）；详见 [`frontend/README.md`](../frontend/README.md)。

### 未来目录建议（前后端分离）
> 目标：前端只负责渲染与输入；后端负责权威逻辑、时间推进与实时同步；两者通过 HTTP/WebSocket 通信。

1. `frontend/`（前端）
   - `frontend/public/`：不参与打包的静态资源（favicon 等）
   - `frontend/src/`
     - `network/`：HTTP（`httpClient`、`api`）、WebSocket（`wsClient`）、`types`
     - `state/`：`gameStore`（订阅 `petStateDelta` 等）
     - `game/`：`GardenScene`、`createGame`（Phaser）
     - `styles/`：`tokens.css`、`app.css`
     - `main.ts`：鉴权 → 领宠 → 离线摘要 → 花园 UI + Phaser + WS（周期 1 闭环）
   - 周期 2+ 可将叠层拆分为 `components/`、`pages/`；通用 UI 参考图仍建议复用根目录 `assets/ui/`

2. `backend/`（后端）
   - `backend/src/cute_cat/`
     - `api/`：HTTP 路由（`routes/`：auth、me、pets、gardens、health）、依赖与统一错误体
     - `auth/`：JWT、密码哈希（bcrypt）
     - `realtime/`：`garden_ws.py`（WebSocket）、`garden_hub.py`（内存房间）
     - `game/`：时间换算、被动衰减、`actions`、`offline` 摘要规则
     - `services/`：宠物状态协调（如 `pet_state`）
     - `persistence/`：SQLAlchemy 模型、会话工厂
     - （预留）`shop/`、`events/`、`ai/`：周期 2+ 按设计补齐
   - `backend/alembic/`：数据库迁移
   - `backend/tests/`：`pytest`（健康检查、注册/领宠/快照/离线摘要；内存 SQLite）
   - `backend/README.md`：安装、迁移、运行、测试
   - `backend/config/`（可选）：后续放 YAML/JSON 可调参数（商店、治疗、成长阈值）

3. `shared/`（共享）
   - `shared/types/`：WebSocket 消息结构、DTO、枚举（动作类型、事件类型、宠物类型等）
   - `shared/constants/`：双方共同使用的关键常量（如 `GameDay` 与时间换算）

4. `doc/`
   - 保持你现在的用法：`项目设计文档.md` 与 `项目目录说明.md` 分别用于“规格”和“目录用途”

5. `assets/`（共享资源）
   - `assets/ui/`：前端 UI 像素 mockup 参考图
   - 实现后建议逐步扩展为：
     - `assets/sprites/`：宠物/道具/动画素材
     - `assets/ui-assets/`：UI 贴图与字体资源

6. `infra/`（可选）
   - 部署脚本、Dockerfile、环境变量示例、CI 配置等（后续再补）

### 技术栈建议（本项目固定口径）
1. 后端（Python）
   - `FastAPI`（HTTP + WebSocket）
   - `Pydantic`（协议消息与参数强校验）
   - `uvicorn`（开发/部署）
   - 数据存储：**MySQL 8**（开发可用本机）；ORM `SQLAlchemy 2.0` + `Alembic` 迁移
   - 可选：`Redis`（JWT refresh 旋转/吊销、限流、多 worker 下 WebSocket 连接映射）
   - AI：`LangChain` + 阿里云 `DashScope`（通义千问，如 `qwen-plus`）；密钥仅环境变量
   - 鉴权：JWT access + refresh（详见 [API-后端与前端对接.md](API-后端与前端对接.md)）
2. 前端（HTML UI 叠层 + Phaser 2D）
   - 构建与类型：`Vite + TypeScript`
   - 渲染与动画：`Phaser 3`（花园场景/像素动画/坐标与命中检测）
   - UI：HTML/CSS 叠层（状态条、按钮、弹窗、提示条）
   - 网络：WebSocket client（发送 `ClientToServer`，处理 `ServerToClient`）
3. 共享（协议一致性）
   - `shared/`：放消息结构/枚举/关键常量，确保前后端字段名一致（类型可以用“手动同步/导出”两种方式，MVP 可手动先跑通）。
